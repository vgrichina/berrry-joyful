# berrry-joyful UI Design

## Design Philosophy

- **Permissions First**: Show permissions screen immediately on launch if not granted
- **Controls Front & Center**: Main UI focuses on control configuration
- **Logs Secondary**: Debug logs hidden in a collapsible panel or separate window
- **Clear Visual Feedback**: Show connection status, active mode, battery level
- **Accessibility**: Large buttons, clear labels, keyboard shortcuts

## Design Notes

### Color Scheme (for implementation)
- **Background**: Dark mode (#1E1E1E) / Light mode (#FFFFFF)
- **Primary**: Blue (#007AFF) - for active states
- **Success**: Green (#34C759) - for connected, granted
- **Warning**: Orange (#FF9500) - for not granted, battery low
- **Error**: Red (#FF3B30) - for errors, disconnected
- **Text**: White/Black with proper contrast

### Typography
- **Headers**: SF Pro Display, 18pt, Bold
- **Body**: SF Pro Text, 13pt, Regular
- **Buttons**: SF Pro Text, 13pt, Medium
- **Monospace**: SF Mono, 11pt (for logs)

### Spacing
- Section padding: 20px
- Element spacing: 12px
- Button padding: 10px 20px

### Interaction States
- **Hover**: Slight highlight (+10% brightness)
- **Active**: Blue background with white text
- **Disabled**: 40% opacity, no interaction
- **Focus**: Blue outline ring

### Accessibility
- All buttons have keyboard shortcuts
- High contrast mode support
- Screen reader labels on all controls
- Minimum touch target: 44x44pt
